Embedded real-time systems often operate under strict deadlines. Missing these deadlines can be detrimental to system performance and correct functioning. At the same time, designers must balance budgets on power, area and cost. An effective approach to designing robust and responsive systems is to utilize multiprocessors \cite{WOLF2023453}. Multiprocessor systems contain multiple \acrlong{PE}s (\acrshort{PE}), that can be optimized for specific functions, performing tasks in parallel. It is well known that the use of simpler units running in parallel can provide the same performance at a lower power and cost \cite{sarpeshkarUniversalPrinciplesUltra2012, WOLF2023453}.

In addition to programmable \acrshort{PE}s, e.g. \acrlong{CPU}s (\acrshort{CPU}), which provide great flexibility, multiprocessor systems can achieve enormous performance improvements by the use of specialized hardware accelerators \cite{WOLF2023453}. One such example is the \acrfull{GPU}, which has become a staple of modern personal computers. \acrshort{FPGA}s allow designers to design custom logic at low costs, as compared to the higher performance and costly \acrlong{ASIC}s (\acrshort{ASIC}). Most \acrshort{FPGA}s also come at the added benefit of being reconfigurable, which provides flexibility for remotely deployed systems.

\autoref{fig:fpga} shows the layout of a typical FPGA. By use of \acrlong{LUT}s (\acrshort{LUT}) and \acrlong{FF}s (\acrshort{FF}), \acrlong{CLB}s (\acrshort{CLB}) can be configured to provide any logic function. Units are mounted in a large routing matrix, allowing custom routing between the \acrshort{CLB}s. These two mechanisms provide the core functionality of the \acrshort{FPGA}, and allow the implementation of any logic design. In order to increase efficiency in area, power and performance, it is common to add additional, specialized hardware units. \acrlong{DSP}s (\acrshort{DSP}) provide common arithmetic functions, while embedded \acrfull{BRAM} provides efficient data storage. The hardware resource utilization of a design for \acrshort{FPGA} is typically measured in terms of the amount of \acrshort{LUT}s, \acrshort{FF}s, \acrshort{DSP}s, and \acrshort{BRAM} used.

Another way of utilizing parallelization to increase performance is pipelining. By separating data independent stages of the calculation with registers, known as pipeline registers, the clock frequency can be increased while still issuing new data for processing every cycle. This approach is widely used in hardware designs, for example in \acrshort{CPU}s. The clock frequency is limited by the longest path between any two pipeline registers. For implementations of the issue 2 algorithm, the aforementioned data dependencies create problems when trying to achieve high degrees of pipelining.

Compared to software design, hardware design comes at increased complexity in terms of design and development. A typical design starts at a microarchitectural level before the design is converted to the \acrfull{RTL} using languages like VHDL, Verilog and Systemverilog. \acrshort{RTL} code is then synthesized to a gatelevel netlist which, through a process called \acrfull{PnR}, is mapped to a specific \acrshort{FPGA} or for \acrshort{ASIC}. To ensure correct functioning of the final result, extensive verification must be performed at all steps in the design process.

To alleviate the challenges in long design time of \acrshort{RTL} code, a new design flow has emerged named \acrfull{HLS}. The advantage of using the \acrshort{HLS} approach is reduced development time, as the algorithm can be described in a high-level language like C or C++, which is then converted to \acrshort{RTL} \cite{barriosHardwareImplementationCCSDS2021}. However, the tools for such a conversion create suboptimal results when compared to writing the implementation directly in \acrshort{RTL} using languages like VHDL or Systemverilog, giving the designer fine-grained control over optimizations.

The satellites of the \acrshort{hypso} project use \acrlong{MPSoC}s (\acrshort{MPSoC}) from AMD (formerly Xilinx), namely the Zynq chips \cite{grotteOceanColorHyperspectral2022}. These \acrshort{COTS} embed two ARM processing units, a simple \acrshort{GPU}, and several useful peripherals including \acrlong{ADC}s (\acrshort{ADC}), timers and support for various communication protocols. The system runs the Linux operating system, providing tools for management of complex tasks and threads. Additionally, the chips include embedded \acrfull{PL}, equivalent to an embedded \acrshort{FPGA}. Embedding the \acrshort{PL} directly in the \acrshort{MPSoC} allows custom logic to directly access the main data buss and other systems on the chip, including \acrfull{DMA} for fast and \acrshort{CPU} independent memory accesses. These features give great benefits in terms of efficiency in speed and power as compared to separate \acrshort{FPGA}s.
